Theory & Algorithm

Classical image inversion computes f'(i,j) = 255 − f(i,j) for every pixel, requiring O(N) sequential operations. Quantum inversion exploits NEQR superposition to apply this transformation to all pixels with a constant-depth gate sequence.

Classical Inversion

For each pixel (i,j) in an N×M image:

f'(i,j) = 255 − f(i,j) ∀ (i,j) ∈ [0,N) × [0,M) Complexity: O(N·M) operations

Quantum Inversion — Bitwise NOT via Pauli-X

In NEQR, f(i,j) is stored as an 8-bit binary number in the value register. Applying X (bit-flip) to every value qubit computes bitwise NOT:

|f(i,j)⟩ → X⊗8 |f(i,j)⟩ = |f̄(i,j)⟩ = |255 − f(i,j)⟩ The superposition state means ALL pixels are inverted simultaneously: X⊗8 |I⟩ = 1/√N Σ_{i,j} |255 − f(i,j)⟩ |i⟩ |j⟩ Additional gate complexity: O(1) — exactly 8 X gates
✓ Pauli-X is its own inverse (X² = I), so the inversion is perfectly reversible — a key quantum property with no classical analogue.

Implementation

The inversion module (src/inversion.py) takes any NumPy image array. Images are automatically resized to the nearest power-of-2 dimensions for NEQR compatibility.

import inversion
import numpy as np
from PIL import Image

img = np.array(Image.open("photo.jpg").convert("L"), dtype=np.float64)

result = inversion.run(img)

# result keys:
#   original, quantum, classical  — NumPy arrays (H×W float64)
#   mse, psnr, ssim               — quality metrics vs classical
#   qubits, depth, gate_count     — circuit properties
#   elapsed, image_size           — runtime info

print(f"MSE:  {result['mse']:.6f}")
print(f"PSNR: {result['psnr']:.2f} dB")
print(f"SSIM: {result['ssim']:.6f}")
ℹ Images of any size are supported. The encoder automatically determines the required number of qubits: n_qubits = 8 + 2·⌈log₂(max(H,W))⌉. For a 256×256 image this is 8 + 16 = 24 qubits.
