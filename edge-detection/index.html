Theory & Algorithm

Classical edge detectors — Sobel, Canny, Laplacian-of-Gaussian — convolve every pixel with a kernel, requiring $O(N \cdot k)$ operations for image size $N$ and kernel size $k$. The quantum approach computes all pixel differences in a single circuit pass by exploiting NEQR superposition and a quantum cyclic shift on the position register.

NEQR Quantum Image State

The input image is encoded as a superposition over all pixel positions in the NEQR basis:

$$|I\rangle \;=\; \frac{1}{\sqrt{2^n}} \sum_{i=0}^{2^n - 1} |f(i)\rangle \otimes |i\rangle \tag{1}$$

Here $|f(i)\rangle$ is the 8-qubit value register encoding grayscale intensity $\in [0, 255]$, and $|i\rangle$ spans the position register ($n_\text{row} + n_\text{col}$ qubits).

Quantum Cyclic Shift Operator

The position register is incremented modulo $2^n$ using a quantum ripple-carry adder (multi-controlled X gates). This shifts every pixel at $(i, j)$ to $(i, j+1)$ in superposition — effectively a cyclic horizontal shift of the whole image:

$$S_\text{col}\,|I\rangle \;=\; \frac{1}{\sqrt{2^n}} \sum_{i,j} |f(i,j)\rangle \otimes |i\rangle \otimes \bigl|(j+1) \bmod 2^{n_c}\bigr\rangle \tag{2}$$
$$S_\text{row}\,|I\rangle \;=\; \frac{1}{\sqrt{2^n}} \sum_{i,j} |f(i,j)\rangle \otimes \bigl|(i+1) \bmod 2^{n_r}\bigr\rangle \otimes |j\rangle \tag{3}$$

Gate count for cyclic shift on $k$ qubits: $\displaystyle G_k = \sum_{i=1}^{k} C^i\text{-}X = \frac{k(k+1)}{2}$ gates. For an 8×8 image ($k = 3$): exactly 6 multi-controlled gates per axis.

Edge Extraction via Pixel Difference

Decoding both the original and the shifted statevectors gives two pixel grids. Their difference is the first-order finite-difference approximation to the image gradient — an edge map. The combined magnitude is the Euclidean gradient norm:

$$h_\text{edge}(i,j) \;=\; \bigl|f(i,j) - f(i,\,j+1)\bigr| \qquad \text{(horizontal)} \tag{4}$$
$$v_\text{edge}(i,j) \;=\; \bigl|f(i,j) - f(i+1,\,j)\bigr| \qquad \text{(vertical)} \tag{5}$$
$$\text{combined}(i,j) \;=\; \sqrt{h_\text{edge}^2 + v_\text{edge}^2},\quad \text{normalised to } [0,255] \tag{6}$$
✓ This approach is inspired by the Quantum Hadamard Edge Detection (QHED) framework (Yao et al., 2017, Phys. Rev. X 7, 031041), which exploits quantum difference operators to detect intensity discontinuities across all pixels in a single circuit evaluation.

Algorithm Steps

1. Encode image in NEQR → $|I\rangle$

2. Statevector sim → decode original pixel grid

3. Apply $S_\text{col}$ → decode horizontal shift

4. Apply $S_\text{row}$ → decode vertical shift

5. Compute $h_\text{edge}$, $v_\text{edge}$, and combined gradient map

6. Normalise and compare vs Sobel / Laplacian

Complexity Analysis

NEQR encoding: $O(2^n)$ gates (one-time cost)

Cyclic shift per axis: $O(n^2)$ gates, $n = \log_2 N$

Statevector decode: $O(2^n)$ classical post-processing

Classical Sobel: $O(N \cdot k)$ — sequential per pixel

Quantum shift circuit: $O(\log_2^2 N)$ — logarithmic scaling

Circuit Design

The full circuit is the NEQR encoding layer followed by the quantum cyclic shift applied to either the column or row register. Two separate circuit evaluations produce the horizontal and vertical edge maps independently.

Qiskit edge-detection circuit (12 qubits, 4×4 representative image). Hadamard gates prepare the position registers in uniform superposition; the NEQR block encodes pixel intensities via multi-controlled X gates; the cyclic shift block — a CX followed by an X — implements $|\,b_1 b_0\rangle \to |(b_1 b_0 + 1) \bmod 4\rangle$ on the column register. Repeating with row register gives the vertical edge map.

Conceptual Layout — NEQR + Horizontal Cyclic Shift

val[0..7] ─────(NEQR encode)────────────── ← value register (unchanged) row[0..n] ─┤H├─ ─────────────────────────── ← row position (unchanged) col[0] ─┤H├─ ───────────┤X├──────────── ← LSB always flips col[1] ─┤H├─ ───────── ──●───┤X├────── ← CX(col[0], col[1]) col[2] ─┤H├─ ───────── ──●───●───┤X├─ ← MCX(col[0:2], col[2]) └── NEQR encoding ──┘ └─── cyclic +1 col ───┘
ℹ The cyclic increment is a quantum ripple-carry adder with wraparound. For a 3-qubit register it maps $|b_2 b_1 b_0\rangle \to |(b_2 b_1 b_0 + 1) \bmod 8\rangle$. Each multi-controlled X (MCX) gate is decomposable into $O(k^2)$ elementary single- and two-qubit gates on real hardware.

Implementation

The edge detection module (src/edge_detection.py) accepts any NumPy grayscale image. It runs three statevector simulations — original, column-shifted, and row-shifted — and returns both the quantum edge maps and classical baselines in a single result dictionary. 

import edge_detection
import numpy as np
from PIL import Image

img = np.array(Image.open("photo.jpg").convert("L"), dtype=np.float64)

result = edge_detection.run(img)

# result keys:
#   original       — input image (resized to 2ⁿ×2ⁿ, float64)
#   quantum_h      — horizontal edge map (column-wise differences)
#   quantum_v      — vertical edge map   (row-wise differences)
#   quantum        — combined magnitude: √(h²+v²), normalised [0,255]
#   sobel          — classical Sobel gradient magnitude
#   laplacian      — classical Laplacian-of-Gaussian edge map
#   corr_sobel     — Pearson r: quantum vs Sobel
#   corr_laplacian — Pearson r: quantum vs LoG
#   qubits, depth, gate_count, elapsed, image_size

print(f"Corr vs Sobel:     {result['corr_sobel']:.4f}")
print(f"Corr vs Laplacian: {result['corr_laplacian']:.4f}")
print(f"Qubits used:       {result['qubits']}")
print(f"Image size:        {result['image_size']}")

Cyclic Shift Subroutine

def _cyclic_increment(qc, reg):
    # Ripple-carry adder: |b⟩ → |(b+1) mod 2^k⟩
    n = len(reg)
    for i in range(n - 1, -1, -1):
        if i == 0:
            qc.x(reg[0])        # LSB always flips
        else:
            qc.mcx(reg[:i], reg[i])  # carry propagation

Running Both Axes

def _run_shifted(enc, axis):
    qc = enc.build_circuit()
    if axis == "col":
        start = enc.n_val + enc.n_row
        reg = list(range(
            start, start + enc.n_col))
    else:  # row axis
        start = enc.n_val
        reg = list(range(
            start, start + enc.n_row))
    _cyclic_increment(qc, reg)
    sv = _SIM.run(transpile(qc, _SIM))
    return enc.decode_statevector(sv)
ℹ Images of any size are accepted. The NEQR encoder automatically pads to the nearest power-of-2 square — e.g. a $100 \times 120$ image becomes $128 \times 128$. Qubit count grows logarithmically: a $256 \times 256$ image uses $8 + 2 \times 8 = 24$ qubits; a $1024 \times 1024$ image needs only 28.

Background & References

Quantum image processing emerged as a research field in the 2010s, offering exponential parallelism by encoding entire images into a single quantum superposition. Edge detection was one of the first operations shown to benefit from this approach.

Why Quantum Edge Detection?

Edges carry the most perceptually salient information in an image. Classical detectors (Sobel, Canny, LoG) scale as $O(N \cdot k)$ per frame — expensive for high-resolution video. On gate-based quantum hardware, the QHED approach reduces the per-pixel operator to $O(\log_2^2 N)$ shift gates, making real-time quantum edge detection feasible at scale once fault-tolerant hardware is available.

Current classical simulation of quantum circuits is limited to small images due to the $2^n$ statevector memory. The quantum advantage is realised on future hardware where this is not the bottleneck.

Relation to Classical Operators

The quantum cyclic difference operator is equivalent to a circular convolution with the kernel $[1,\,-1]$ along each axis — a first-order finite difference. Sobel uses a $3 \times 3$ kernel incorporating neighbouring pixel weights for noise robustness; the quantum version uses exact nearest-neighbour differences, making it more sensitive to high-frequency detail.

On real quantum hardware (shot-based measurement), the result would require statistical reconstruction from many shots. The statevector simulation here gives the exact quantum-mechanical result — free of shot noise.
